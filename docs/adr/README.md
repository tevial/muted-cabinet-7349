# Architecture Decision Records

Use ADRs for decisions that are expensive or difficult to reverse.

Examples:

- Framework and runtime selection.
- Database and persistence strategy.
- Authentication and authorization model.
- Hosting and deployment architecture.
- Multi-tenant boundaries.
- Integration ownership.
- Eventing, queueing, or workflow engines.
- Security and compliance decisions.

Use [ADR Template](../templates/adr.md).
